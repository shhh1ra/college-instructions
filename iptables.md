# Общая структура работы с iptables

## Основные цепочки:
- **INPUT** (входящие пакеты)
- **OUTPUT** (исходящие пакеты)
- **FORWARD** (пересылаемые пакеты)

`Пакет проходит правила сверху вниз -> первое совпадение решает судьбу`
****
## Базовая структура команды:
```bash
iptables -A INPUT -p tcp --dport 22 -J ACCEPT
```
Разберем команду:
- -A INPUT -> добавить правило в INPUT.
- -p tcp -> какой протокол использовать.
- --dport 22 -> порт назначения.
- -j ACCEPT -> действие, в этом случае разрешить.

`Примечание номер 1: все флаги команды чувствительны к регистру.`

`Примечание номер 2: на практике один раз возникла ситуация, когда пакет iptables есть, но он не находится системой при вводе команды, решается с помощью добавления /sbin/ в начало каждой команды.`
****
## Основные действия:
- **ACCEPT** - разрешить пакет
- **DROP** - отбросить пакет **тихо**
- **REJECT** отклонить пакет **с ответом**
- **MASQUERADE** - NAT
****
## Основные команды iptables:
- Посмотреть текущие правила:
```bash
iptables -L -n -v
```
- Сохранить написанные правила:
```bash
iptables-save >> /etc/sysconfig/iptables
```
