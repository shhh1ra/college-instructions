# Процесс создания стенда.
## Требуемое ПО:
- **Vmware Workstation**
- **Virtual Network Editor (поставляется в комплекте с Vmware Workstation)**
- **Образ Alt Server (Я выбрал 10.4 чтобы иметь практику на нем)**
****
# Настройка виртуальных адаптеров:
## Настройка адаптера "провайдера" для ISP.
**После запуска Vmware Workstation нажимаем по кнопке `edit` в левом верхнем углу и заходим в `Virtual Network Editor`. Сначала получаем права администратора.**

![picture](https://github.com/shhh1ra/college-instructions/blob/main/images/stand-creation/getting-admin-rights.png)

**Далее нужно первоначально настроить `VMnet0` адаптер, тип адаптера нас интересует bridged, и лучше заранее указать какую сетевую карту будет использовать VMware, у меня это Realtek 8852BE.**

![picture](https://github.com/shhh1ra/college-instructions/blob/main/images/stand-creation/provider-network.png)
****
## Настройка адаптера ISP <-> HQ-RTR:
- **Сначала нужно создать новый адаптер, VMware спросит какой адаптер вы хотите создать, нужно будет выбрать `VMnet2`. Нужно выбрать тип адаптера `Host-only`, снять галочку на `локальный DHCP сервис для виртуальных машин`, задать `Subnet IP` и `Subnet mask`.**

![picture](https://github.com/shhh1ra/college-instructions/blob/main/images/stand-creation/add-new-adapter.png)

- **По заданию у сегмента ISP <-> HQ-RTR адрес сети `192.168.4.0 или 192.168.40.0` на выполнение задания это не влияет, у меня будет адрес `192.168.40.0`.**

![picture](https://github.com/shhh1ra/college-instructions/blob/main/images/stand-creation/settings-for-new-adapter.png)

****
## Остальные адаптеры:
**Я не буду разбирать так же досканально создание остальных адаптеров, они все делаются по одному шаблону, но приложу таблицу как правильно их настроить.**

| **Название адаптера** | **Subnet IP** | **Subnet mask** |
| --------------------- | ------------- | --------------- |
| ISP-HQ                | 192.168.40.0  | 255.255.255.0   |
| ISP-BR                | 192.168.50.0  | 255.255.255.0   |
| HQ-LAN                | 192.168.100.0 | 255.255.255.0   |
| BR-LAN                | 192.168.200.0 | 255.255.255.0   |
